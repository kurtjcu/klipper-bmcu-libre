<objective>
Add direction_invert config option to BmcuChannel so V2.2 hardware users can fix the reversed
motor wiring polarity. When direction_invert is True, the Klipper extra swaps FWD/REV in DIR
commands sent to firmware. No firmware changes needed.

Purpose: On V2.2 hardware, BMCU_DIR FWD ejects filament instead of feeding it inward. This
config option lets users correct the polarity at the Klipper layer without recompiling firmware.

Output: Modified bmcu_feeder.py with direction_invert support, 5 new tests, updated config docs.
</objective>

```python
def _cmd_dir(self, gcmd):
    ch_id = gcmd.get_int('CHANNEL', minval=0, maxval=3)
    direction = gcmd.get('DIR')
    if direction not in ('FWD', 'REV'):
        raise gcmd.error("BMCU: DIR must be FWD or REV, got '%s'" % direction)
    if ch_id not in self._channels:
        gcmd.respond_info("BMCU: channel %d not configured" % ch_id)
        return
    self._serial.send("DIR %d %s\n" % (ch_id, direction))
```

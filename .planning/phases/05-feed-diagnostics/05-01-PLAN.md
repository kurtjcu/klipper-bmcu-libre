<objective>
Add per-channel feed distance tracking, cumulative stall counting, and a BMCU_RESET_FEED GCode command so users can monitor filament consumption and stall events from Mainsail/Fluidd and reset counters between print jobs.

Purpose: Users currently have no way to see how far each channel has fed or how many stalls occurred. This plan exposes feed_mm_since_reset and stall_count via Moonraker's get_status() and provides BMCU_RESET_FEED to zero both counters (suitable for START_PRINT macros).

Output: Updated bmcu_feeder.py with diagnostics fields, BMCU_RESET_FEED command, updated get_status(); tests for all three DIAG requirements; config documentation.
</objective>

def _dispatch_status_line(self, line):
    for m in _STATUS_FIELD_RE.finditer(line):
        ch_id = int(m.group(1))
        if ch_id not in self._channels:
            continue
        ch = self._channels[ch_id]
        old_state = dict(ch.state)
        ch.state.update({
            'filament_present': m.group(2) == '1',
            'motor_running':    m.group(3) == '1',
            'speed':            int(m.group(4)),
            'direction':        m.group(5),
            'feed_mm':          float(m.group(6)),
            'mag_status':       m.group(7),
        })
        self._check_events(ch, old_state)
